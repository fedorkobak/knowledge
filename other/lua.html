Lua#

Lus is an interpreted programming language that is popular to be language for creating plugins for different software.

This notebook uses xeus-lua kernel to show the aspects of lua.

Functions#

The functions in lua are really intuitive. However, there are some important details to be aware of. Check the Functions page.

There are two syntaxes for defining a function, both of which are represented in the following example:

function hello(name)
    print("Hello!", name)
end

hello("Lua")

Hello! Lua

hello2 = function(name)
    print("Hello!", name)
end

hello2("Lua2")

Hello! Lua2

Tables#

Lua offers the table datatype to store data. This object can represent both sequences and mappings.

Check more details in the specific tables page.

The following cell defines the table1 as table:

table1 = {
    'the first element',
    key1 = 'value1',
    ['key2'] = 'value2'
}
table1

{ the first element, 
  key1 = value1,
  key2 = value2 
}

Here are table that contains:

  • The “the first element” string as the first element of the sequence.

  • Under the key1 hides value value1.

  • Under the key2 hides value value2.

By accessing the first index of the table, you retrieve value that is defined as an element of the table:

table1[1]

the first element

There are different ways to access the values under the mapping:

table1["key1"]

value1

table1.key2

value2

Flow control#

The flow control constructions available in lua are listed in the following table:

Construct

Description

if ... then ... end

Conditional execution.

if ... then ... else ... end

Conditional execution with alternative branch.

if ... then ... elseif ... then ... else ... end

Multi-branch conditional.

while ... do ... end

Pre-condition loop.

repeat ... until

Post-condition loop (runs at least once).

for i = start, stop [, step] do ... end

Numeric loop.

for ... in ... do ... end

Generic loop using iterators.

break

Exits the innermost loop.

goto label

Jumps to a label within the same function.

::label::

Defines a label for goto.

return

Exits a function and optionally returns values.

do ... end

Creates a local scope block.

Check mode detailed explanation with some examples in the dedicated Flow control page.

Standard library#

Check the Standard libraries reference page.

The following talbe show the most essential functions of the lua standard library:

Name

Description

print

Prints values to standard output, converting them to strings.

type

Returns the type of a value ("nil", "number", "string", "table", etc.).

pairs

Returns an iterator for traversing all key-value pairs in a table.

ipairs

Returns an iterator for traversing the integer-indexed elements of an array-like table.

next

Returns the next key-value pair in a table; used internally by pairs.

tonumber

Converts a value to a number if possible.

tostring

Converts a value to its string representation.

pcall

Calls a function in protected mode, catching errors instead of raising them.

require

Loads and returns a module, caching it for future use.

error

Raises an error and stops normal execution unless caught by pcall or xpcall.

Modules#

The modles in lua can be imported by standard library require function.

The module is simply a lua file that returns a value. Typically, this is table named M that contains references to all the objects the module provides.

Consider the module lua_files/example.lua:

M = {}

M.some_function = function()
    print("This is function from module")
end

return M

The code that imports that module:

example_module = require("lua_files.example")
example_module

This is function from module

The usage of some_function has the expected behaviour:

example_module.some_function()

This is function from module
