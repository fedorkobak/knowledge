docker built - сборка образа

Создание docker образа

Создают “Dockerfile” (так и называется), который описывает конструируемый образ.

Для того, чтобы собрать образ используется комманда docker build.

  • -t - задать имя и тэг нашего образа;
  • -f - задать имя файла из которго собирается образ (но лучше всегда исопльзовать “dockerfile”);
%%bash
docker build -t custom_ubuntu:test -f my_first_dockerfile .

Базовые инструкции

Тут пречислены самые базовые инструкции которые могут быть указаны в dockerfile.

  • FROM - указыват на основе какого базового образа будет собираемый;
  • WORKDIR - задает рабочую директорию;
  • COPY <компьютер> <контейнер> скопировать файл в образ;
  • RUN позволяет запустить некоторые команды при сборке образа.

Пример образа использующего все эти команды. Предствален в файле my_first_dockerfile (образы так называть нельзя).

Команды контейнеру при запуске образа

Следующие две комманды будут выполняться в образе в момент его запуска.

ENTRYPOINT

К комманде записанной такио образом будут дописаны аргументы переденные при запуске docker. Так, собирая образ по файлу (Dockerfile_ENTRYPOINT):

FROM ubuntu:20.04
COPY script.sh script.sh
ENTRYPOINT [ "bash" ]

И приписывая туда “scrip.sh”, мы при запуске в комбинации получим комманду bash script.sh

%%bash
docker run --rm entrypoing_example script.sh

CMD

Этой же комманде приписок сделать не получиться. Если мы добавляем аргумент при запуске контейнера. Но комманда записанная в CMD просто удалится. Так при запуске контейнера собранного на остове следующего файла:

FROM ubuntu:20.04
COPY script.sh script.sh
CMD [ "bash" ]

bash был перезаписан на ls поэтому в результате были выведены папки рабочей дирректории.

Комбинация

С помощью комбинации, в ENTRYPOINT можно задать обязательно выполняющуюся часть комманды а в CMD ту, которая будет изменятся в завимимости от входа. Так образ созданный по Dockerfile вида:

FROM ubuntu:20.04
ENTRYPOINT ["echo"]
CMD ["wow"]

(Dockerfile_ENTRYPOIND_CMD)

Будет работать следующим образом.

%%bash
docker run --rm entrypoind_cmd_example
docker run --rm entrypoind_cmd_example hello
wow
hello

Особенности интерактивного режима

Следует иметь ввиду, что если исопользуются комманды ENTRYPOINT или CMD то можно не попасть в интерактивный режим как это привычно происходит в случае использования опций -it, потому как комманды туда не всегда заложенные не ожидают в себя ввода. Так например контейнер из примера выше:

Детали которые следует иметь ввиду

Если файл создается в каком либо слое то он в образе навстегда

Удаление файла не уменьшает размер образа. То есть docker захеширует даже файл который затем будет удален таким образом файл будет сидеть в образе.

Для примера рассмотрим два dockerfile

Первый файл dockerfile_rm_example1:

    FROM ubuntu:20.04
    COPY script.sh

Второй файл dockerfile_rm_example2:

    FROM ubunty:20.04
    COPY script.sh
    RUN rm script.sh

docker commit

Позволяет состояние некоторго запущенного контейнера сохранить как образ. Вот например в запущеннй контейнер ubuntu я символически в нем обновляю пакеты, сохраняю под новым именем и просматриваю список образов в котром, конечно находится новый образ.

docker run --rm --name commit_example -itd ubuntu
docker exec commit_example apt-get -qq update -qq

docker stop commit_example
